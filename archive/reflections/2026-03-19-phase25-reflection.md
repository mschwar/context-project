# Phase 25 Reflection — Production Hardening (Pre-1.0.0 Assessment)

**Date:** 2026-03-19
**Branch:** `feat/phase24-manifest-trust`
**Tests:** 426 passing (7 new)

## Successes

- **Rate limit architecture redesigned.** Retry-After header support, 60s backoff ceiling, 5 retry attempts, configurable concurrency. The previous design (3 retries, 4s max backoff, 4 parallel workers) was architecturally incapable of surviving Anthropic's per-minute token limits.
- **Mid-level budget enforcement works.** `threading.Event` stops workers within a depth level. Previously, submitting a depth level to the executor was all-or-nothing — a single 100-file directory could overshoot budget by 9x.
- **Exit code semantics are clean.** Exit 2 for partial success (budget exhausted) means scripted loops and CI pipelines can distinguish "made progress, run again" from "something broke."
- **`--until-complete` auto-resume works end-to-end.** The 390-directory grant portfolio completed in a single invocation with Ollama. The incremental skip logic + auto-resume loop means large trees "just work" without manual re-runs.
- **All 7 fixes landed on one branch with no regressions.** 419 existing tests untouched, 7 new tests added, all 426 pass.

## Friction Points

- **Stress test revealed model quality gap, not infrastructure gap.** The `cydonia-24b-remote` (24B, Q4_K_M) model produced empty file summaries and generic "dirname directory." purpose sentences across all 390 directories. ctx's infrastructure handled everything correctly — the model just didn't produce useful output.
- **No pre-flight model quality check exists.** ctx has no way to detect before processing 390 directories that the local model will return garbage. A single-file probe would catch this in seconds.
- **`ctx check --verify` catches empty file summaries but not generic directory summaries.** Every directory purpose was "dirname directory." — clearly placeholder text — but none were flagged.

## Observations

- The grant portfolio is ~95% binary files (.docx, .xlsx, .pdf, .pptx). Only 17 directories had text files requiring LLM summarization. Binary markers (`[binary: ext, size]`) are generated by ctx itself, so they're always correct regardless of model quality.
- `local_batch_fallbacks: 20` confirms the adaptive batching safety net triggered appropriately during the run.
- The `tokens_total: 0` in most manifests (373/390) is correct — binary-only directories don't invoke the LLM.
- Backward compatibility for `.ctxconfig` files using the old `batch_size` key works correctly.

## Suggestions

1. **Pre-flight model quality check** — Send one sample file to the configured model before starting bulk processing. Verify the response contains substantive text. Warn and offer to abort if the model returns empty/placeholder output.
2. **Minimum summary length threshold** — If an LLM returns empty or whitespace-only text for a file summary, retry once or emit a warning. Currently these pass silently and cause `--verify` failures.
3. **Local model compatibility matrix** — Document which models are tested and produce useful summaries. Include model name, size, quantization level, and quality assessment.
