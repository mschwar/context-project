Front-loading compact, structured metadata + summaries (treating files as self-contained conversations, imagine each file as an end-to-end self-contained user/agent/LLM conversation. all the information in the file is compacted and placed in the contents field of 'contents.txt'), agents can make smart decisions at the directory level before diving deeper

a file system native hierarchical manifest system for coarse-to-fine retrieval ("like robots.txt but for providing local file system context instead of restricting access.

core: filesystem native (no external vector DB required) with per-directory files, conversation compaction, and agent-first navigation. lightweight, portable, git-friendly, human-inspectable, and works offline. Optimized for agent swarms crawling repos, project folders, chat logs, or knowledge bases.
