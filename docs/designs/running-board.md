## Key Review Findings

### Spec Updates Required
1. **Section 2 wording:** Change "one record per cycle" to "one record per invocation" for `--until-complete`
2. **Acceptance criteria #9:** Update from "net +20" to "net +35" tests
3. **File change summary:** Add MCP server file, update test counts
4. **Global write:** Add file-lock retry (3 attempts, 100ms backoff) to `_update_global`

### Architecture Decision
- `--until-complete` writes one record per invocation (not per cycle) — cache counters accumulate across cycles naturally since CachingLLMClient is reused

### Data Integrity
- Global file writes use file-lock retry (3 attempts, 100ms backoff) to handle concurrent writes from parallel refreshes
